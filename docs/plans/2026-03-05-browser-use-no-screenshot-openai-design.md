# Browser-Use Fork 设计文档：默认禁用截图链路 + 统一外部 OpenAI 配置

**日期：** 2026-03-05  
**状态：** 已确认方案 A（可实现，不在本轮改代码）  
**适用分支：** 当前 fork 分支（登录态页面稳定性优先）

---

## 1) 背景与问题定义

当前分支目标是让 agent 在“已人工登录”的页面上稳定执行读取与操作，不要求 agent 自主完成登录流程。现有调试记录显示核心阻塞在截图流水线：即使禁用视觉输入，DOM 状态获取链路仍可能触发截图事件，并在复杂 SPA 上被 `captureScreenshot` 超时阻塞，导致状态提取失败、任务循环停滞。与此同时，模型接入层存在多种 OpenAI 风格 API 使用需求（官方 OpenAI、兼容网关、自建转发），希望形成统一配置入口，避免每个 provider 分别绕路配置。  

本设计遵循两个原则：第一，**把登录态可用性当作一等目标**，默认关闭截图及其后续事件链，减少与页面渲染复杂度耦合；第二，**把外部 OpenAI 兼容能力产品化**，通过统一配置模型封装 `api_key/base_url/model` 及结构化输出兼容策略，提升可移植性与可维护性。该设计不追求一次性“最通用重构”，而是优先实现可预测行为和最小风险迁移路径。

---

## 2) 目标与非目标

### 目标
- 新增显式开关，语义为“禁用截图流水线”，并设为默认开启（即默认禁用截图）。
- 禁用时不触发截图动作，不发布/不等待截图相关事件，不阻塞 DOM 状态提取。
- 统一外部 OpenAI 配置模型，覆盖官方 OpenAI 与 OpenAI-compatible 网关。
- 保持登录态使用方式简单：允许人工预登录后复用 profile/CDP，不要求 agent 模拟登录。

### 非目标
- 不在本轮设计中处理 CAPTCHA、反爬绕过、隐身指纹增强。
- 不强制切换现有默认 LLM 生态，仅增加统一配置入口与兼容语义。
- 不在本轮承诺任何“自动登录能力”增强。

---

## 3) 方案 A 架构设计（推荐落地）

### 3.1 截图链路治理：默认关闭 + 显式开关

引入配置项（命名可在实现时微调）：
- `disable_screenshot_pipeline: bool = True`

设计语义：
- `True`（默认）：系统在状态采集流程中直接走“无截图路径”；不触发截图事件；相关 watchdog 不进入截图执行分支；DOM 提取结果不因截图缺失而失败。
- `False`：保持现有截图流程，兼容需要视觉证据的场景。

关键行为约束：
- 任何 `BrowserStateRequest` 到 DOM 状态返回的主路径，必须在 `disable_screenshot_pipeline=True` 下完全无截图依赖。
- 禁用截图不应影响点击、输入、滚动、提取等核心动作执行。
- 关闭截图后，日志中应出现可观测标记（例如 `screenshot_pipeline=disabled`），便于问题定位。

### 3.2 外部 OpenAI 统一配置模型

新增统一配置视图（概念模型）：
- `provider`: `openai` | `openai_compatible`
- `api_key`
- `base_url`（官方可为空并回落默认）
- `model`
- `supports_json_schema: bool | None`（可选显式声明）
- `dont_force_structured_output: bool | None`（兼容不支持 `json_schema` 的端点）

装配原则：
- 官方 OpenAI：`provider=openai`，允许省略 `base_url`。
- 兼容网关：`provider=openai_compatible`，必须可传 `base_url`。
- agent 初始化阶段统一解析该模型，再映射到底层 `ChatOpenAI` 或兼容客户端参数，避免在业务代码多处分叉判断。

---

## 4) 数据流与失败语义

### 4.1 无截图路径数据流
1. Agent 发起状态请求。  
2. DOM 采集服务直接执行 DOM/可交互元素提取。  
3. 跳过截图事件发布与等待。  
4. 返回状态给 LLM 进行下一步决策。  

### 4.2 失败语义（重点）
- 截图禁用时，`screenshot` 字段可为空或被显式标记为 `disabled`，但不得触发异常。
- 如果实现中某处仍要求截图非空，应统一改为“可空安全处理”并附日志。
- 外部 OpenAI 端点若不支持 `response_format=json_schema`，统一由配置层下沉到兼容策略，不在执行时反复报错后回退。

---

## 5) 兼容性与迁移策略

- **向后兼容：** 提供开关，保留旧截图路径；仅默认值改为关闭。
- **风险控制：** 对依赖截图的历史流程，明确要求显式开启开关。
- **迁移成本：** 既有脚本多数只需增加或确认配置；对于登录态任务可直接受益于默认无截图。

---

## 6) 验收标准（Definition of Done）

- 默认配置下执行登录态页面任务时，不触发截图链路且不出现截图等待阻塞。
- DOM 提取在复杂 SPA 下可持续推进多步决策。
- 官方 OpenAI 与兼容网关使用同一配置模型均可初始化并执行任务。
- 兼容端点在不支持 `json_schema` 时可通过统一配置稳定运行。
- 所有行为在文档中有明确示例与故障排查说明。

---

## 7) 决策冻结（已确认）

- **开关落位：** 同时支持 `Agent` 与 `Browser` 两层配置，默认值均为 `disable_screenshot_pipeline=True`。  
- **优先级规则：** `Agent` 显式传参优先于 `Browser` 默认配置（就近覆盖，避免行为歧义）。  
- **接口暴露范围：** Python API 与 CLI 同步暴露该开关，默认行为一致。  
- **screenshot action 禁用语义：** 返回结构化提示型结果（non-fatal no-op），不抛异常、不阻塞主流程，便于 agent 继续执行。
